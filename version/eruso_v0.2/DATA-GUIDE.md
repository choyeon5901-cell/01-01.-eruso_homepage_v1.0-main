## 🚀 빠른 시작

### 방법 1: add-data.html 사용 (권장)

1. **파일 열기**
   ```
   프로젝트 폴더 / add-data.html
   ```
   브라우저에서 `add-data.html` 파일을 엽니다.

2. **데이터 추가**
   - 상단 탭에서 추가할 데이터 유형 선택
   - 필수 정보 입력
   - "추가" 버튼 클릭

3. **확인**
   - "데이터 조회" 탭에서 등록 확인
   - 실시간 통계 확인

### 방법 2: test.html 사용

샘플 데이터를 빠르게 추가하려면 `test.html`을 사용하세요.

## 🏥 병원 정보 추가

### 필수 입력 항목

| 항목 | 설명 | 예시 |
|------|------|------|
| 병원명 | 의료기관 이름 | 조치원 중앙병원 |
| 주소 | 병원 소재지 전체 주소 | 세종특별자치시 조치원읍 군청로 93 |
| 전화번호 | 병원 대표 전화 | 044-860-1000 |
| 면허번호 | 의료기관 면허번호 | H2024-001 |
| 담당자명 | 병원 담당자 이름 | 김의사 |
| 담당자 이메일 | 로그인용 이메일 | doctor@hospital.com |
| 담당자 전화번호 | 담당자 연락처 | 010-1234-5678 |

### 입력 절차

1. **add-data.html 열기**
2. **"병원 추가" 탭 선택**
3. **정보 입력**
   - 모든 필수 항목(*) 입력
   - 주소는 상세하게 입력
4. **"병원 추가" 버튼 클릭**
5. **성공 메시지 확인**

### 자동 생성 항목

병원 정보 추가 시 다음 항목이 자동으로 생성됩니다:
- ✅ 병원 계정 (hospitals 테이블)
- ✅ 담당자 사용자 계정 (users 테이블, role: hospital)
- ✅ 고유 ID (UUID)
- ✅ 등록 일시 (created_at)
- ✅ 활성 상태 (status: active)

### 로그인 방법

병원 추가 후 담당자는 다음과 같이 로그인할 수 있습니다:
1. `index.html`에서 "로그인" 클릭
2. 등록한 담당자 이메일 선택
3. `hospital.html` 포털로 이동

### 데이터 조회 탭

"데이터 조회" 탭에서는 다음 정보를 확인할 수 있습니다:

- 등록일

#### 환자 목록
- 이름
- 이메일
- 전화번호
- 주소
- 상태 (활성/비활성)
- 등록일

### 데이터 새로고침

최신 데이터를 확인하려면:
1. "데이터 조회" 탭 선택

## ❓ 자주 묻는 질문

### Q1. 병원/약국 추가 시 담당자 계정은 어떻게 생성되나요?
**A:** 병원/약국 정보 입력 시 담당자 정보를 함께 입력하면 자동으로 사용자 계정이 생성됩니다. 담당자는 입력한 이메일로 로그인할 수 있습니다.

### Q2. 같은 이메일로 여러 계정을 만들 수 있나요?
**A:** 기술적으로는 가능하지만, 권장하지 않습니다. 각 사용자는 고유한 이메일을 사용해야 로그인 시 혼란을 방지할 수 있습니다.

### Q3. 환자 주소는 얼마나 상세하게 입력해야 하나요?
**A:** 배송을 위해 동/호수까지 상세하게 입력하는 것이 좋습니다.
예: "세종특별자치시 조치원읍 서부로 123, 101동 501호"

### Q4. 입력한 정보를 수정할 수 있나요?
**A:** 현재 버전에서는 add-data.html을 통한 수정 기능이 없습니다. 데이터베이스에 직접 접근하거나 개발자에게 요청해야 합니다.

### Q5. 데이터를 삭제할 수 있나요?
**A:** 현재 버전에서는 add-data.html을 통한 삭제 기능이 없습니다. 필요시 개발자에게 요청해야 합니다.

### Q6. 면허번호 형식은 어떻게 되나요?
**A:** 특정 형식은 없지만, 구분을 위해 다음과 같이 사용하는 것을 권장합니다:
- 병원: H2024-001, H2024-002...
- 약국: P2024-001, P2024-002...

### Q7. 데이터가 저장되지 않아요!
**A:** 다음을 확인하세요:
1. API 연결 상태 확인 (`diagnose.html` 사용)
2. 필수 항목을 모두 입력했는지 확인
3. 브라우저 콘솔에서 오류 메시지 확인
4. Genspark 플랫폼에 배포되어 있는지 확인

### Q8. 로컬에서 사용할 수 있나요?
**A:** 제한적으로 가능합니다:
- `fix-404.html`에서 로컬 모드 활성화
- 데이터는 브라우저 LocalStorage에 저장
- RESTful API 기능은 사용 불가
- 완전한 기능을 위해서는 Genspark 배포 필요

### Q9. 테스트 데이터는 어떻게 추가하나요?
**A:** `test.html`을 열고 "샘플 데이터 추가" 버튼을 클릭하면 테스트용 데이터가 자동으로 생성됩니다.

### Q10. 데이터를 내보낼 수 있나요?
**A:** 향후 업데이트에서 CSV/Excel 내보내기 기능이 추가될 예정입니다.

## 🎯 권장 워크플로우

### 1단계: 병원 및 약국 추가
```
1. add-data.html 열기
2. 병원 정보 추가 (1개 이상)
3. 약국 정보 추가 (1개 이상)
4. 데이터 조회 탭에서 확인
```

### 2단계: 환자 추가
```
1. 환자 정보 추가 (테스트용 환자 생성)
2. 데이터 조회 탭에서 확인
```

### 3단계: 전체 워크플로우 테스트
```
1. 병원 로그인 → 처방전 발행
2. 환자 로그인 → 처방전 확인
3. 약국 로그인 → 접수 → 조제 → 배송
4. 환자 로그인 → QR 인증 수령
5. 관리자 로그인 → 전체 현황 확인
```
