DB 동기화 가이드

로컬과 배포 환경의 데이터베이스가 다를 때

문제: 다른 DB를 사용 중

로컬 DB ID: 2ffad277-fb19-4a57-be62-003ab454a10c-db

Cloudflare DB ID: a4ac0e30-bade-41ae-937e-d36f22ef2478

⚠️ 완전히 다른 데이터베이스를 사용하고 있습니다!

현재 상황

✅ 로컬/미리보기 환경

DB ID: 2ffad277-fb19-4a57-be62-003ab454a10c-db 상태: 데이터 있음 ✅ ↓ - 사용자 데이터 ✅ - 병원/약국 데이터 ✅ - 처방전 데이터 ✅ - 배송 정보 ✅

👉 test.html에서 추가한 샘플 데이터가 여기에 저장되어 있습니다

❌ Cloudflare 배포 환경

DB ID: a4ac0e30-bade-41ae-937e-d36f22ef2478 상태: 비어있음 ❌ ↓ - 사용자 데이터 ❌ - 병원/약국 데이터 ❌ - 처방전 데이터 ❌ - 배송 정보 ❌

👉 완전히 새로운 빈 데이터베이스입니다

왜 이런 일이 발생하나요?

Genspark는 환경별로 별도의 데이터베이스를 생성합니다:

  • 개발/미리보기 환경 → 개발용 DB (2ffad277...)
    여기서 테스트하고 샘플 데이터를 추가하는 곳
  • 프로덕션/배포 환경 → 배포용 DB (a4ac0e30...)
    실제 사용자가 접속하는 배포된 사이트

📌 이것은 정상적인 동작입니다! 개발 데이터와 프로덕션 데이터를 분리하는 것이 보안상 더 안전합니다.

해결 방법 (3가지)

✅ 방법 1: 배포 환경에서 직접 데이터 추가 (권장)

가장 간단하고 안전한 방법입니다.

  1. 배포된 사이트로 이동
    https://your-site.genspark.app
  2. test.html 페이지 열기
    https://your-site.genspark.app/test.html
  3. "샘플 데이터 추가" 버튼 클릭
  4. 성공 메시지 확인 ✅
  5. 포털 페이지로 이동하여 데이터 확인

👍 장점: 간단하고 빠름, 배포 DB에 직접 추가

📤 방법 2: CSV 파일로 일괄 등록

대량의 데이터를 한 번에 추가할 때 유용합니다.

  1. CSV 파일 준비
    • sample_hospitals.csv (병원 데이터)
    • sample_patients.csv (환자 데이터)
    • 또는 직접 작성한 CSV 파일
  2. 배포된 사이트의 bulk-upload.html 열기
    https://your-site.genspark.app/bulk-upload.html
  3. CSV 파일 업로드 (드래그 앤 드롭)
  4. 미리보기 확인
  5. "일괄 등록 실행" 버튼 클릭

👍 장점: 대량 데이터 한 번에 등록 가능

📝 방법 3: 개별 수동 추가

소량의 데이터를 정확하게 추가할 때 사용합니다.

  1. 배포된 사이트의 add-data.html 열기
    https://your-site.genspark.app/add-data.html
  2. 병원/약국/환자 정보를 하나씩 입력
  3. "추가" 버튼으로 저장
  4. "데이터 조회" 탭에서 확인

👍 장점: 정확한 데이터 입력, 실시간 확인 가능

중요: DB 마이그레이션은 불가능

로컬 DB의 데이터를 배포 DB로 직접 이동할 수는 없습니다.

  • Genspark의 DB는 환경별로 완전히 분리됨
  • 보안상의 이유로 직접 마이그레이션 불가
  • 대신 배포 환경에서 직접 데이터를 추가해야 함

💡 따라서 위의 3가지 방법 중 하나를 선택하세요!

patient.html, hospital.html 등에서 데이터 확인

추가 도움말

  • 배포 환경: 실제 사용자가 사용하는 프로덕션
  • 각 환경마다 별도로 데이터를 추가해야 합니다
