배포 URL 확인 가이드

"이 페이지에 연결할 수 없습니다" 오류 해결

현재 오류

DNS_PROBE_FINISHED_NXDOMAIN 홉... 이 페이지에 연결할 수 없습니다. your-site.genspark.app 오타가 있는지 확인합니다.

⚠️ 이것은 도메인을 찾을 수 없다는 의미입니다.

원인 분석

❌ 가능한 원인들:

  • 배포가 완료되지 않음
    → Publish 탭에서 배포가 진행 중이거나 실패했을 수 있음
  • 잘못된 URL 사용
    → "your-site" 부분이 실제 프로젝트 이름으로 변경되지 않음
  • 배포 URL을 확인하지 않음
    → Publish 탭에서 실제 배포된 URL을 복사하지 않고 예시 URL 사용
  • DNS 전파 대기 중
    → 방금 배포했다면 몇 분 정도 기다려야 할 수 있음

올바른 배포 URL 찾는 방법

1

Genspark으로 돌아가기

이 프로젝트가 있는 Genspark 탭/창으로 돌아갑니다.

2

Publish 탭 클릭

화면 상단에서 "Publish" 탭을 찾아 클릭합니다.

💡 Publish 탭이 없다면 아직 배포하지 않은 것입니다.

3

배포 상태 확인

다음 중 하나의 상태를 확인하세요:

✅ 배포 완료:

배포된 URL이 표시됩니다 (예: https://abc123def.genspark.app)

⏳ 배포 중:

진행 표시줄이 보입니다. 완료될 때까지 기다려주세요.

❌ 배포 실패:

에러 메시지가 표시됩니다. "Deploy" 버튼을 다시 클릭하세요.

4

실제 URL 복사

배포가 완료되면 실제 배포된 URL이 표시됩니다.

예시:

https://project-abc123def456.genspark.app https://jochiwon-delivery-xyz789.genspark.app https://smart-medicine-delivery-123.genspark.app

👆 이런 형식의 URL이 표시됩니다. 복사 버튼을 클릭하세요.

5

새 탭에서 열기

복사한 URL을 브라우저 주소창에 붙여넣고 Enter를 누릅니다.

아직 배포하지 않은 경우

Publish 탭이 없거나 "Deploy" 버튼만 보인다면:

  1. Publish 탭 클릭
  2. "Deploy" 또는 "Publish" 버튼 클릭
  3. 배포 완료까지 대기 (보통 1-3분)
  4. 배포 완료 후 URL 표시됨
  5. URL 복사 후 접속

URL 형식 비교

❌ 잘못된 예시 (작동하지 않음)

https://your-site.genspark.app https://my-project.genspark.app https://example.genspark.app

👆 이런 URL은 예시일 뿐입니다. 실제로 작동하지 않습니다!

✅ 올바른 예시 (실제 배포 URL)

https://project-7a8b9c0d1e2f3g4h.genspark.app https://jochiwon-medicine-delivery-xyz123.genspark.app

👆 Publish 탭에서 제공하는 실제 URL을 사용해야 합니다!

방금 배포한 경우

배포가 막 완료되었다면 DNS 전파를 기다려야 할 수 있습니다:

  • 일반적으로 1-5분 정도 소요
  • 브라우저 캐시 삭제: Ctrl + Shift + Delete
  • 시크릿 모드로 시도: Ctrl + Shift + N
  • 5분 후에도 안 되면 배포 URL 재확인

배포 URL 확인 체크리스트

여전히 문제가 있다면

다음을 확인하세요:

  1. 배포가 실제로 완료되었는지 확인
    • Publish 탭에서 "Deployment successful" 메시지
    • URL이 표시되어 있음
  2. 올바른 URL을 사용하는지 확인
    • "your-site" 같은 예시 URL이 아닌지
    • 실제 배포 URL을 복사했는지
  3. 브라우저 캐시 문제
    • Ctrl + Shift + Delete로 캐시 삭제
    • 시크릿 모드로 접속 시도
  4. 다시 배포 시도
    • Publish 탭에서 "Deploy" 버튼 다시 클릭
    • 배포 완료 후 새 URL 확인

배포 URL로 접속 성공 후

사이트가 정상적으로 로드되었다면:

  1. test.html로 이동하여 샘플 데이터 추가
    → 배포 DB는 비어있으므로 데이터를 추가해야 합니다
  2. 포털 페이지에서 기능 테스트
  3. 문제가 있다면 deployment-troubleshoot.html로 진단
