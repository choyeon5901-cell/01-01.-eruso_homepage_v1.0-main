# 조치원 스마트 약배송 서비스

세종특별자치시 조치원읍을 위한 통합 의약품 배송 관리 시스템입니다. 처방전 발행부터 약 배송까지의 전 과정을 디지털화하여 효율적으로 관리합니다.

## 🚀 빠른 시작

### 🎯 어디서 시작해야 할지 모르겠다면?
**→ `start.html`을 여세요!** 모든 도구와 포털 링크가 한 곳에 있습니다.

### ⚡ 배송 상태 문제 즉시 해결 (가장 빠른 방법!)
**문제**: 처방전 상태는 "완료"인데 배송 상태가 "대기중"으로 표시됨

**해결**: 
1. `auto-fix.html`을 브라우저에서 엽니다
2. "지금 바로 수정하기" 버튼을 클릭합니다
3. 자동으로 모든 배송 상태가 수정됩니다! ✅

**파일 위치**: 프로젝트 폴더의 루트에 있습니다
```
조치원-스마트-약배송/
├── auto-fix.html  ← 이 파일입니다!
├── patient.html
├── hospital.html
└── ...
```

- **병원 로그인**: test.html에서 "조치원 중앙병원" 선택 → 로그인
- **약국 로그인**: test.html에서 "조치원 건강약국" 선택 → 로그인
- **관리자 로그인**: test.html에서 "관리자" 선택 → 로그인

또는 `index.html`에서 로그인 버튼을 클릭하여 시작할 수 있습니다.

## 🆕 데이터 추가 도구 (add-data.html & bulk-upload.html)

### 📝 기능 소개
**add-data.html**은 병원, 약국, 환자 정보를 쉽고 빠르게 추가할 수 있는 통합 관리 도구입니다.
**bulk-upload.html**은 CSV 파일을 사용하여 대량의 데이터를 한 번에 등록할 수 있습니다.

### ✨ 주요 기능

#### 📋 개별 추가 (add-data.html)
1. **병원 정보 추가**
   - 병원명, 주소, 전화번호
   - 의료기관 면허번호
   - 담당자 정보 (이름, 이메일, 전화번호)
   - 자동으로 사용자 계정 생성

2. **약국 정보 추가**
   - 약국명, 주소, 전화번호
   - 약국 면허번호
   - 약사 정보 (이름, 이메일, 전화번호)
   - 자동으로 사용자 계정 생성

3. **환자 정보 추가**
   - 이름, 이메일, 전화번호
   - 주소 정보
   - 생년월일 (선택사항)

4. **데이터 조회 및 관리**
   - 등록된 모든 병원, 약국, 환자 목록 조회
   - 실시간 통계 대시보드
   - 데이터 새로고침 기능

#### 📤 일괄 등록 (bulk-upload.html)
1. **CSV 템플릿 다운로드**
   - 병원, 약국, 환자별 CSV 템플릿 제공
   - 예시 데이터 포함
   - Excel 호환 (UTF-8 BOM)

2. **약국 데이터 조회**
   - 등록된 약국 목록 실시간 조회
   - 약국명, 주소, 전화번호, 면허번호, 상태 표시
   - 모달 팝업으로 깔끔한 UI 제공

3. **파일 업로드**
   - 드래그 앤 드롭 지원
   - CSV 파일 자동 파싱
   - 데이터 유형 자동 감지

4. **데이터 미리보기**
   - 업로드 전 데이터 확인
   - 처음 5개 행 미리보기
   - 총 행 수 표시

5. **일괄 등록 실행**
   - 실시간 진행 상황 표시
   - 성공/실패 건수 표시
   - 오류 상세 정보 제공

### 🚀 사용 방법

#### 개별 추가
1. `add-data.html` 파일을 브라우저에서 엽니다
2. 상단 탭에서 추가할 데이터 유형을 선택합니다
3. 필수 정보를 입력합니다
4. "추가" 버튼을 클릭합니다
5. "데이터 조회" 탭에서 등록된 정보를 확인합니다

#### 일괄 등록
1. `bulk-upload.html` 파일을 브라우저에서 엽니다
2. 원하는 데이터 유형의 CSV 템플릿을 다운로드합니다
3. Excel 또는 텍스트 편집기로 데이터를 입력합니다
4. CSV 파일을 저장합니다 (UTF-8 인코딩 권장)
5. 파일을 업로드 영역에 드래그하거나 클릭하여 선택합니다
6. 미리보기를 확인한 후 "일괄 등록 실행" 버튼을 클릭합니다

#### 📱 포털에서 환자 일괄등록 (관리자/병원)
1. **관리자 포털**에서:
   - `admin.html` → "사용자 관리" 탭 → "환자 일괄등록" 버튼 클릭
   - bulk-upload.html로 이동하여 환자 CSV 파일 업로드

2. **병원 포털**에서:
   - `hospital.html` → "새 처방전 발행" 카드 → "환자 일괄등록" 버튼 클릭
   - bulk-upload.html로 이동하여 환자 CSV 파일 업로드

3. **약국 데이터 조회**:
   - bulk-upload.html에서 "약국 데이터 조회" 버튼 클릭
   - 등록된 모든 약국 정보를 실시간으로 확인
   - 약국명, 주소, 전화번호, 면허번호, 활성 상태 표시

#### 🆕 포털별 등록 기능 (2024년 2월 14일)

**환자 포털**에서 병원/약국 등록:
1. `patient.html` → "병원 / 약국 등록" 섹션
2. "병원 등록하기" 또는 "약국 등록하기" 버튼 클릭
3. 개별 등록: add-data.html로 이동하여 정보 입력
4. 일괄 등록: "일괄 등록" 버튼 클릭 → bulk-upload.html로 이동

**병원 포털**에서 환자/약국 등록:
1. `hospital.html` → "환자 / 약국 등록" 섹션
2. "환자 등록하기" 또는 "약국 등록하기" 버튼 클릭
3. 개별 등록: add-data.html로 이동하여 정보 입력
4. 일괄 등록: "일괄 등록" 버튼 클릭 → bulk-upload.html로 이동

**스마트 연동**:
- URL 파라미터로 역할 전달 (`?role=patient`, `?role=hospital`)
- 자동으로 해당 역할에 맞는 탭 선택
- 역할별 맞춤 안내 메시지 표시

### 📊 통계 대시보드
화면 상단에서 다음 정보를 실시간으로 확인할 수 있습니다:
- 등록된 병원 수
- 등록된 약국 수
- 등록된 환자 수
- 병원 담당자 수

### 💡 사용 팁
- 모든 필수 항목(*)은 반드시 입력해야 합니다
- 병원/약국 추가 시 담당자 계정이 자동으로 생성됩니다
- 담당자는 해당 이메일로 로그인할 수 있습니다
- 데이터 조회 탭에서 등록 상태를 확인할 수 있습니다

```
   ↓
5. [약국] 배송 완료 (completed)
```

### 🆕🔥🔥 로컬과 배포 DB가 다를 때 (중요!)

**가장 흔한 배포 혼란입니다!**

**증상**:
- 로컬/미리보기: 데이터가 잘 보임 ✅
- 배포 사이트: 데이터가 없음 ❌

**원인**:
```
로컬 DB ID:     2ffad277-fb19-4a57-be62-003ab454a10c-db
배포 DB ID:     a4ac0e30-bade-41ae-937e-d36f22ef2478
                ↓
              완전히 다른 DB!
```

**이유**:
- Genspark는 **환경별로 별도의 데이터베이스**를 생성합니다
- 개발 환경 DB ≠ 프로덕션 환경 DB
- 보안상 더 안전하지만, 각 환경마다 별도로 데이터를 추가해야 함

**즉시 해결**:
1. `db-sync-guide.html` 페이지를 엽니다
2. 3가지 해결 방법 중 선택:
   - ✅ **방법 1 (권장)**: 배포 사이트에서 test.html → "샘플 데이터 추가"
   - 📤 **방법 2**: bulk-upload.html에서 CSV 일괄 등록
   - 📝 **방법 3**: add-data.html에서 개별 추가

**중요**: 로컬 DB 데이터를 배포 DB로 직접 이동할 수 없습니다. 배포 환경에서 직접 데이터를 추가해야 합니다!

### 문제: 약국 포털에서 완료 건수가 0으로 표시됨
**원인**: 
1. 약국 ID로 필터링할 때 완료된 처방전이 포함되지 않음
2. 완료 탭 배지가 업데이트되지 않음

**해결 방법** (✅ 수정 완료):
- pharmacy.html의 `loadPrescriptions` 함수가 `pharmacy_id`로 모든 상태의 처방전을 불러오도록 개선
- `updateStats` 함수에 `completedBadge` 업데이트 추가
- 완료 탭에 배지 추가 (`<span class="badge badge-success" id="completedBadge">0</span>`)
- 페이지 새로고침 시 정상 표시

**검증 방법**:
1. `verify.html` 페이지를 엽니다
2. "데이터 검증" 버튼을 클릭하여 완료 건수를 확인합니다
3. 완료 처방전이 없으면 "완료 처방전 생성" 버튼을 클릭합니다
4. 약국 포털을 새로고침하여 완료 건수가 표시되는지 확인합니다

### 문제: QR 코드가 생성되지 않음
**원인**: 배송 정보를 찾지 못함

**해결 방법** (이미 수정됨):
- pharmacy.html의 `loadDeliveries` 함수가 처방전 ID로도 배송 정보를 조회하도록 개선
- 약국 포털에서 "배송 시작" 버튼 클릭 시 QR 코드 자동 생성
- 처방전 상세보기에서 QR 코드 이미지 확인 가능

### 문제: 배송 상태가 "대기중"으로만 표시됨
**원인**: 약국에서 상태를 변경해도 배송 정보가 업데이트되지 않음

**해결 방법** (이미 수정됨):
- 배송 시작 시: `delivery.status = 'in_transit'` 자동 업데이트
- 배송 완료 시: `delivery.status = 'delivered'` 자동 업데이트
- 정상적으로 프로세스를 진행하면 배송 상태가 자동으로 업데이트됨

**기존 데이터 수정** 🆕:
1. `sync.html` 페이지를 엽니다
2. "상태 확인" 버튼을 클릭하여 불일치 데이터를 확인합니다
3. 불일치가 있으면 "배송 정보 동기화 실행" 버튼을 클릭합니다
4. 처방전 상태에 맞춰 배송 상태가 자동으로 업데이트됩니다

**상태 매핑 규칙**:
- 처방전 `issued` (발행됨) → 배송 `pending` (대기중)
- 처방전 `received` (접수완료) → 배송 `pending` (대기중)
- 처방전 `preparing` (조제중) → 배송 `pending` (대기중)
- 처방전 `delivering` (배송중) → 배송 `in_transit` (배송중)
- 처방전 `completed` (완료) → 배송 `delivered` (배송완료)
