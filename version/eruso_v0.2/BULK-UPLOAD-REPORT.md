## 🌟 주요 기능

### bulk-upload.html의 핵심 기능

#### 1. CSV 템플릿 다운로드
- ✅ **병원 템플릿**: 병원명, 주소, 전화번호, 면허번호, 담당자 정보
- ✅ **약국 템플릿**: 약국명, 주소, 전화번호, 면허번호, 약사 정보
- ✅ **환자 템플릿**: 이름, 이메일, 전화번호, 주소, 생년월일
- ✅ **예시 데이터 포함**: 각 템플릿에 샘플 데이터 1-2개 포함
- ✅ **Excel 호환**: UTF-8 BOM 추가로 한글 깨짐 방지

#### 2. 파일 업로드
- ✅ **드래그 앤 드롭**: 파일을 끌어다 놓기만 하면 업로드
- ✅ **클릭 업로드**: 업로드 영역 클릭으로 파일 선택
- ✅ **자동 파싱**: CSV 파일 자동 읽기 및 파싱
- ✅ **데이터 유형 감지**: 헤더를 분석하여 병원/약국/환자 자동 감지
- ✅ **파일 정보 표시**: 파일명, 크기, 데이터 행 수 표시

#### 3. 데이터 미리보기
- ✅ **표 형식**: 깔끔한 표 형식으로 데이터 표시
- ✅ **처음 5개 행**: 업로드할 데이터의 처음 5개만 미리보기
- ✅ **전체 행 수**: 전체 데이터 행 수 표시
- ✅ **확인 후 실행**: 미리보기 확인 후 등록 실행

#### 4. 일괄 등록 실행
- ✅ **실시간 진행률**: 프로그레스 바로 진행 상황 표시 (%)
- ✅ **진행 메시지**: 현재 처리 중인 행 번호 표시 (예: 5 / 10)
- ✅ **서버 부하 방지**: 각 요청 사이에 100ms 딜레이
- ✅ **오류 처리**: 실패한 행의 오류 메시지 및 데이터 표시
- ✅ **성공/실패 카운트**: 총 성공 건수 및 실패 건수 표시

#### 5. 결과 표시
- ✅ **성공 통계**: 녹색 카드에 성공 건수 표시
- ✅ **실패 통계**: 빨간색 카드에 실패 건수 표시
- ✅ **오류 목록**: 실패한 행의 상세 정보 및 오류 메시지
- ✅ **재시도 버튼**: "다시 등록하기" 버튼으로 페이지 새로고침
- ✅ **데이터 조회**: "데이터 조회" 버튼으로 add-data.html 이동

## 🎯 사용 시나리오

### 시나리오 1: 10개 병원 일괄 등록

```
1단계: 템플릿 다운로드
   bulk-upload.html 열기
   → "병원 템플릿" 다운로드 버튼 클릭
   → hospitals_template.csv 다운로드

2단계: 데이터 입력
   Excel로 hospitals_template.csv 열기
   → 10개 병원 정보 입력
   → UTF-8 인코딩으로 저장

3단계: 파일 업로드
   업로드 영역에 파일 드래그
   → 파일 정보 확인 (10개 행)
   → 미리보기에서 처음 5개 확인

4단계: 일괄 등록
   "일괄 등록 실행" 버튼 클릭
   → 진행률 0% → 100% 확인
   → 성공: 10개, 실패: 0개

5단계: 확인
   "데이터 조회" 버튼 클릭
   → add-data.html에서 병원 목록 확인
```

### 시나리오 2: 50명 환자 일괄 등록

```
1. patients_template.csv 다운로드
2. 50명 환자 정보 입력
3. CSV 파일 업로드
4. 미리보기 확인 (5개만 표시, 총 50개)
5. 일괄 등록 실행
6. 결과 확인: 성공 50개, 실패 0개
7. 환자 포털에서 로그인 테스트
```

### 시나리오 3: 오류 처리

```
상황: 20개 약국 등록 중 3개 오류 발생

1. CSV 파일 업로드 (20개 행)
2. 일괄 등록 실행
3. 결과 화면:
   - 성공: 17개
   - 실패: 3개
4. 오류 목록 확인:
   - 행 5: 이메일 형식 오류
   - 행 12: 전화번호 누락
   - 행 18: 면허번호 중복

6. "다시 등록하기" 클릭
7. 수정된 파일 재업로드
8. 성공: 3개, 실패: 0개
```

- 담당자명: 병원 담당자 이름
- 담당자이메일: 로그인용 이메일 (고유해야 함)
- 담당자전화번호: 담당자 휴대폰 번호

## 💡 사용 팁

### 1. CSV 파일 작성
```
✅ DO (권장)
- Excel에서 템플릿을 열어 수정
- "다른 이름으로 저장" → "CSV UTF-8(쉼표로 분리)"
- 모든 필수 항목 입력
- 이메일은 고유하게 (중복 방지)

❌ DON'T (비권장)
- 메모장에서 직접 작성 (오류 발생 가능)
- ANSI 인코딩으로 저장 (한글 깨짐)
- 빈 행 포함
- 헤더 변경 또는 삭제
```

### 2. 대량 데이터 등록
```
💡 팁:
- 한 번에 100개 이하로 나누어 등록
- 서버 부하를 고려하여 시간 간격을 두고 실행
- 오류 발생 시 해당 데이터만 수정하여 재등록
- 중요 데이터는 백업 후 등록
```

### 3. 오류 방지
```
체크리스트:
□ 이메일 형식 확인 (예: user@example.com)
□ 전화번호 형식 확인 (예: 010-1234-5678)
□ 면허번호 고유성 확인 (중복 방지)
□ 필수 항목 누락 확인
□ 주소 상세 입력 (배송용)
□ UTF-8 인코딩 확인
```

### 4. Excel 저장 방법
```
Excel 2016 이상:
1. 파일 → 다른 이름으로 저장
2. 파일 형식: CSV UTF-8(쉼표로 분리)(*.csv)
3. 저장

Excel 2013 이하:
1. 파일 → 다른 이름으로 저장
2. 파일 형식: CSV(쉼표로 분리)(*.csv)
3. 저장 후 메모장에서 열기
4. 파일 → 다른 이름으로 저장
5. 인코딩: UTF-8
6. 저장
```
